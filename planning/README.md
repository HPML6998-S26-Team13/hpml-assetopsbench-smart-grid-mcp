# Planning Index

Current task truth lives in GitHub Issues/Projects and the authored docs under `docs/`. This directory is intentionally light: active meeting/planning records should be short-lived, dated, and moved to `archive/` after the meeting or decision window passes.

## Current Contents

- [`archive/`](archive/) — historical agendas, prep docs, meeting notes, and action plans. Use these for reconstructing discussion history, not for current PR/issue state.

## Policy

- Canonical current status: GitHub Issues/Projects plus `docs/README.md` and `docs/validation_log.md`.
- Meeting records: dated Markdown, archived after the relevant call.
- Scratch/private planning: keep in the personal class repo, not this team repo.
